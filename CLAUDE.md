### Confidence Score Formula (data-quality based)
`conf = base(0.10) + source_count(0-0.25) + pyramid_type(0-0.25) + note_coverage*0.30`
`confidence_score = min(conf × rating_mult, 0.95)`

| Component | Value | Contribution |
|---|---|---|
| Base | always | 0.10 |
| source_count ≥ 6 | | 0.25 |
| source_count 4-5 | | 0.22 |

| source_count 1 | | 0.05 |
| Real pyramid | not inferred | 0.25 |
| Inferred pyramid | has_inferred_pyramid=True | 0.15 |
| No pyramid | | 0.00 |
| Note coverage | fraction × 0.30 | 0.00–0.30 |

rating_mult: >1000: ×1.05 | 200–1000: ×1.00 | 50–200: ×0.95 | <50: ×0.85

**Phase 6 — What changed and why:**
1. **rating_count backfill** (`scripts/update_ratings.py`) — 61,188 records populated from CSV
   sources. Sauvage EDT: 23,727 ratings → ×1.05 multiplier → confidence 0.663 → 0.819.
2. **100% label blend** — `_generate_labels()` in `ml/model.py` now uses `label_hours` directly
   for labeled perfumes instead of 50/50 blend. Strong recall: 52.7% → 70.7%.
3. **Calibration bug fix** — `calibrate_longevity.py` was calling `predict()` which applied the
   OLD calibrator, then training a new calibrator on already-calibrated values (double-calibration).
   Fixed by adding `skip_calibration=True` parameter to `predict()` and using it in the calibrator
   training loop. New calibrator: raw 3.24–10.0h → label midpoints 2.0–12.0h, MAE 0.81h.

### Remaining Weaknesses
1. **Light recall = 34.8%** — 15/23 light perfumes still predicted as moderate. The calibrator
   maps very light fragrances correctly to 2h, but moderate perfumes can also land in the 2-4h
   range post-calibration. Fix: add more light-labeled perfumes to the training set, or apply
   a separate threshold for the light/moderate boundary.
2. **Strong recall = 70.7%** — 44/150 strong perfumes predicted as moderate. Remaining error
   comes from perfumes with chemistry profiles that overlap the moderate range (e.g., medium-strong
   label = 7.5h, borderline with strong bucket threshold of 8h).
3. **6,034 perfumes** still have rating_count = 0 — these are niche/obscure perfumes not in either
   CSV source. They use ×0.85 confidence multiplier. No further action without additional data.
4. **Dedup bug (same-brand, same-name, different-concentration)** — Sauvage EDT and EDP both
   normalize to `by_brand['dior']['sauvage']`; higher-id record steals the slot. Tracked for
   next import refactor.

### Pyramid Inference (commit feat: note pyramid inference…)
Run `python scripts/infer_pyramids.py` from `backend/`. Idempotent — skips perfumes where
`has_inferred_pyramid=TRUE` already. Algorithm: Jaccard similarity on accord vectors (98-dim
binary, batched numpy matmul, chunks of 1000), top-5 similar reference perfumes, notes
aggregated by similarity-weighted frequency, top-15 notes re-sorted by volatility to assign
top/middle/base (5 each).

Stats: 42,995 pyramids inferred in ~90s, 0 skipped.

## Do Not Do

- **Do not attempt live Fragrantica scraping** — Cloudflare blocks it; this was already tried and abandoned.
- **Do not re-run the full importer without checking** — the import is complete and idempotent, but re-running 70k rows against a remote DB takes ~90 min. Verify `SELECT COUNT(*) FROM perfumes` ≥ 67,222 before deciding to re-run.
- **Do not skip syntax/type checks before committing** — run mypy/pyright on changed files.
- **Do not retrain models unless data has actually changed** — use `python scripts/test_model.py` from `backend/`.
- **Do not re-run calibrate_longevity.py unless models were retrained** — calibrator is tied to the current pkl weights.
- **Do not call `predict()` with the default calibration when training the calibrator** — always pass `skip_calibration=True` in `calibrate_longevity.py`. Failing to do so composes calibrators and breaks inference.
- **Do not re-run update_ratings.py redundantly** — it is idempotent (only updates rating_count=0 records) but scanning 70k CSV rows and 67k DB records takes ~10 min. The backfill is complete.
